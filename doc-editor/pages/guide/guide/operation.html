初识选区
什么是选区？
选区表示在文档中选中的片段。当我们使用鼠标框选了文档中的一段内容时，我们就创建了一个选区，我们也可以把这个选中的效果叫做拖蓝。当我们点击文档的某个位置，并且显示出光标时，我们也创建了一个选区，不过这个选区是一个零宽的。
我们在前面的文档中，已见到过选区的使用，现在我们就选区做进一步了解。我们可以在任意可访问到 Textbus 实例的地方获取到选区的实例。如：
import { Selection } from '@textbus/core'

const selection = textbus.get(Selection)
TypeScript
也可以在任意组件内，通过 useContext 获取到选区的实例：
class YourComponent extends Component {
  setup() {
    const selection = useContext(Selection)
  }
}
TypeScript
一个选区总是会包含开始位置、结束位置，焦点位置和锚点位置。它们在 Textbus 中分别表示为：
selection.startSlot // 开始插槽
selection.startOffset // 开始位置在开始插槽的偏移量

selection.endSlot // 结束插槽
selection.endOffset // 结束位置在结束插槽的偏移量

selection.focusSlot // 焦点插槽
selection.focusOffset // 选区的焦点插槽偏移量

selection.anchorSlot // 锚点插槽
selection.anchorOffset // 选区的锚点插槽偏移量
TypeScript
选区的开始位置总是位于结束位置之前或等于结束位置，而选区的焦点和锚点位置则于我们的框选行为相关，一般来说，锚点为鼠标按下的位置，而焦点是鼠标抬起的位置。
也就是说，当从文档前面向后框选时，开始位置和锚点位置相等，结束位置和焦点位置相等，而从文档后面向前框选时，开始位置和焦点位置相等，而结束位置和锚点位置相等，你可以根据你的需要，灵活的使用开始、结束、焦点和锚点。
插槽与选区的关系
在 Textbus 中，插槽是动态可编辑内容的容器，插槽内可以插入文本，组件，也可以附加样式等。由于我们可以给插槽附加任意多的样式，所以一个插槽的渲染结果将可能是无穷无尽的，Textbus 是怎样确定我们选中的位置呢？
实际上，Textbus 根本不关心渲染的结果，只关注插槽内的内容，即文本和组件。对于文本的格式，则不会影响选区的计算。在设置选区时，我们只需要关注我要选中哪些内容即可，不需要关注渲染的结果的标签是怎么嵌套的。
如图所示，Textbus 选区只在内容上生效，strong 标签的存在并不会影响内容的下标索引。
Textbus 还会忽略组件渲染出的内容，如一个组件渲染出了一大段文本或其它内容，这些内容将是不可选中的，它们将作为一个整体进行插入或删除。
当一个插槽中插入了组件，如果组件没有子插槽，那么光标将在组件的前后移动，而永远不会进入到组件内，无论组件内有没有文本或其它内容。如果组件有子插槽，光标将经过组件到子插槽内容的所有中间节点，会从组件的根节点直接跳到组件子插槽内。
一句话：只有插槽中的内容才是可编辑的，组件渲染的内容是不可用光标操作或编辑的。
设置选区或光标
我们可以在文档渲染完成后，任意设置选区或光标。前面我们已经了解了开始位置、结束位置、焦点位置和锚点位置。相应的，也有对应方法去更改这些位置，默认情况下，这些位置是跟随用户的操作而发生变化，我们也可以在程序中直接设置。
要设置选区位置，我们必需要拿到要设置的插槽。你可以通过某个组件或其它方式获取到需要的插槽实例。需要注意的是，不能设置为负数，也不能大于插槽的长度（length)。如果为负数将视为 0，如果大于插槽的 length，将直接视为插槽的 length。
// 设置选区的开始位置为指定插槽的第 6 位
selection.setStart(slot, 6)
selection.restore()

// 设置选区的结束位置为指定插槽的第 8 位
selection.setEnd(slot, 8)
selection.restore()

// 设置选区的焦点位置为指定插槽的第 3 位
selection.setFocus(slot, 3)
selection.restore()

// 设置选区的锚点位置为指定插槽的第 9 位
selection.setAnchor(slot, 9)
selection.restore()

// 同时设置选区的锚点为指定插槽的第 3 位，焦点为指定插槽的第 6 位
selection.setBaseAndExtent(slot, 3, slot, 6)
selection.restore()

// 同时设置所有位置，即光标位置
selection.setPosition(slot, 12)
selection.restore()
TypeScript
在数据未产生更改时，如果我们移动了光标，浏览器中的光标并不会产生变化，你还需要调用  selection.restore() 方法同步 Textbus 选区到原生选区。
移动光标
Textbus 实现了常见的光标移动操作。如向后移动、向前移动、移动到上一行、移动到下一行、移动焦点等。
// 向前移动光标
selection.toPrevious()

// 向后移动光标
selection.toNext()

// 向上移动光标
selection.toPreviousLine()

// 向下移动光标
selection.toNextLine()

// 向前框选
selection.wrapToBefore()

// 向后框选
selection.wrapToAfter()

// 向上框选
selection.wrapToPreviousLine()

// 向下框选
selection.wrapToNextLine()
TypeScript
全选及取消选区
// 全选
selection.selectAll()

// 取消选择
selection.unSelect()
TypeScript

命令
什么是命令？
命令在 Textbus 中特指 Commander 类下的一系列方法。通过 Commander 类，我们可以更方便的修改文档。你可以在任意能访问到 Textbus 实例的地方通过 get() 方法获取：
import { Commander } from '@textbus/core'

const commander = textbus.get(Commander)
TypeScript
或在组件内通过 useContext() 方法获取。
class YourComponent extends Component {
  setup() {
    const commander = useContext(Commander)
  }
}
TypeScript
在前面的文档中，我们已经使用过 Slot 的 write、insert、delete 等方法，但这些方法只能我们手动去指定位置，然后调用，并且，我们修改过内容后，用户的光标（即选区）并不会自动跟随数据而产生变化，也不会触发组件内的事件。
输入
使用 Commander 修改文档，会自动根据选区执行操作，同时会自动维护选区位置，还会触发组件的一系列事件。
例如，当我们需要向光标位置输入文字时，只需要执行如下代码即可：
commander.write('text')
TypeScript
当执行上 commaner.write() 方法时，就像用户输入一样，会在文档插入内容，并且，光标也会自动跟随我们的更改。
注意：当输入时，如果选区有框选内容，则会选删除框选的内容，再进行输入。
删除
删除有三种情况，即有框选时的删除、无框选时向前删除、无框选时向后删除。当选区为框选状态时，将直接删除选区内的内容，否则，将按照传入的参数判断是向前还是向后删除。
// 向后删除
commander.delete()

// 向前删除
commander.delete(true)
TypeScript
格式应用与清理
我们可以通过 Commander 的格式操作方法，为选区应用格式或清除格式。
// 为选区应用格式
commander.applyFormat(colorFormatter, '#f00')

// 清除格式
commander.unApplyFormat(colorFormatter)

// 清空所有格式
commander.cleanFormats()
TypeScript
属性的应用与清理
我们可以通过 Commander 的属性操作方法，为选区选中的插槽应用属性或清除属性。
你可以把属性简单理解为块级样式。
// 给选中的插槽添加属性
commander.applyAttribute(textAlignAttribute, 'right')

// 清除属性
commander.unApplyAttribute(textAlignAttribute)

// 清除所有属性
commander.cleanAttributes()
TypeScript
格式变换
在文档编辑过程中，简单的调整样式并不能满足需求，我们还需要能对文档中的块进行变换，如段落变更为列表，或者待办事项等。这些功能在 Textbus 中一般是由不同组件构成的，每种组件都会有自己的状态和功能，并且，这些功能并不是 Textbus 内置的，要自动变换它们不是一件容易的事。
/**
 * 组件转换规则
 */
export interface TransformRule {
    /** 目标组件的数据类型 */
    targetType: ContentType;
    /**
     * 创建目标组件新插槽的工厂函数
     * @param from
     */
    slotFactory(from: Component<any>): Slot;
    /**
     * 创建组件状态的工厂函数
     */
    stateFactory(slots: Slot[], textbus: Textbus): Component<any>[];
}
TypeScript
在调用 Commander 的 transform 方法时，需要传入一个转换规则，Textbus 才能自动转换格式。
我们任意内容转段落和列表为例，并且我们假定列表为多插槽组件，段落为单插槽组件。
任意内容转段落：
commander.transform({
  targetType: ContentType.BlockComponent, // 段落为块级组件
  slotFactory(): Slot {
    // 创建组件插槽，生成段落组件
    return new Slot([
      ContentType.Text,
      ContentType.InlineComponent
    ])
  },
  stateFactory(slots: Slot[], textbus: Textbus) {
    return slots.map(slot => new ParagraphComponent(textbus, {slot}))
  }
})
TypeScript
任意内容转列表：
commander.transform({
  targetType: ContentType.BlockComponent, // 列表组件为块级组件
  slotFactory(): Slot {
    // 创建列表项插槽
    return new Slot([
      ContentType.Text,
      ContentType.InlineComponent
    ])
  },
  stateFactory(slots: Slot[], textbus: Textbus) {
    return new ListComponent(textbus, {slots})
  }
})
TypeScript
更多 Commander 的方法请参考 API 下的 Commander 相关文档。
